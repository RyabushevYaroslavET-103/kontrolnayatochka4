Занятие 9. Структуры. Массив структур.

Требования к решению:
1. Использовать функции для каждой подзадачи.
2. Как минимум одно поле должно иметь перечисляемый тип данных
(enum).
3. При описании структуры использовать в качестве одного из полей
структуру.
4. Ввод данных осуществить в виде инициализации массива структур
внутри программы (не с клавиатуры).
5. Алгоритм сортировки – пузырьком.
6. После выполнения следующего занятия выложить решение на github.
Ссылки прикрепить в контрольную точку как результат работы со
структурами и файлами.

Вариант 28.
Определить структурный тип, описывающий компьютерные игры (название игры,
студия разработчик, стоимость подписки, объем аудитории игроков,
поддерживаемые платформы: PC, mobile, gamepad; тип: RPG, стратегия, гонки,
авиасимулятор и др.). Заполнить структурный массив 20-ю записями. Переписать из
исходного массива в другой массив, информацию только о мобильных играх от
Nintendo. Затем новый массив отсортировать по объему аудитории. Вывести все
данные по конкретной игре. Вывести 2 наименее популярных игр. Реализовать
функцию изменения игры. В отдельный массив поместить все мобильные игры, в
которых можно что-то строить. Реализовать вывод отфильтрованных данных в виде
оберточной функции.

Занятие 10. Чтение и запись данных в файл.
Требования к решению:
1. Используем решение задачи своего варианта из занятия 9. Там уже
представлена нужная структура данных.
2. Использовать библиотеку fstream.
3. Оформить код чтения / записи в файл с использованием подпрограмм.
4. Результат выгрузить на github в виде одной ссылки или в виде одного
текстового файла (pdf). В решении представить условие задачи, текст
решения, результат выполнения программы.

женский). Из файла считываются данные и если фамилия совпадает с
очередным элементом массива структур, то в поле pol записывается
соответствующее значение из файла.
Ivanov 0
Ivanova 1
Petrov 0
Sidorova 1
Makarkin 0

Задание 2. Сделать чтение / запись своей структуры в бинарный файл.
